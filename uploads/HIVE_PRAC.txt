APACHE HIVE ACT AS AN ABSTRACTION OVER APACHE HADOOP MAP REDUCE.

FACEBOOK WHO DEVELOPED APACHE HIVE.

APACHE HIVE WORKS ON THE TOP OF APACHE HADOOP.

APACHE HIVE HAS PREREQUISITE : HADOOP & JAVA

DEFAULT RDBMS WHICH HIVE SUPPORTS :  DERBY

DEFAULT LOCATION OF HIVE TABLES : /user/hive/warehouse

COMMAND TO PRINT AVAILABLE DATABASES : show databases

COMMAND TO PRINT AVAILABLE TABLES : show tables

COMMAND TO ADD COMMENT TO DATABASE : CREATE DATABASE ABC COMMENT "xyz"

PRIMITIVE DATA TYPES IN HIVE : INT,BIGINT,STRING,BOOLEAN

COMPLEX DATA TYPES IN HIVE : ARRAY MAP STRUCT

DDL COMMANDS SUPPORTED IN HIVE : CREATE SHOW DROP DESCRIBE TRUNCATE ALTER USE

FILE STORE THE HADOOP LOCATION : hive-env.sh

METASTORE : STORES METADATA ABOUT TABLES (TABLE_NAME/COLUMN INFO/PRIVILEGES)

PARTITIONING : PARTITIONED BY

set hive.exec.dynamic.partition = true;

BUCKETING : CLUSTERED BY

ALTER COMMAND IN HIVE : RENAME, ADD AND DROP COL. 

CHANGE : RENAME COL NAME AND DATA TYPE
MODES OF OPERATION : LOCAL AND MAP REDUCE
LOCAL MODE : WHEN WE HAVE SMALLER DATASET
MAP REDUCE : WHEN WE HAVE LARGER DATASET
INTERACTIVE AND NON INTERACTIVE MODE : 
INTERACTIVE : WORKING ON HIVE SHELL
NON INTERACTIVE : WORKING ON SCRIPTS
INTERNAL AND EXTERNAL : INTERNAL IS DEFAULT
FOR MAKING USE OF EXTERNAL TABLE, EXTERNAL AND LOCATION KEYWORD IS USED
CREATE TABLE SYNTAX:
CREATE TABLE IF NOT EXISTS employee ( eid int, name String,
salary String, destination String)
ROW FORMAT DELIMITED
FIELDS TERMINATED BY ‘\t’
LINES TERMINATED BY ‘\n’
STORED AS TEXTFILE;
